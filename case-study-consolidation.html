Bupa 2022–Present Design Systems

Component Library Consolidation

Unifying 4 fragmented design libraries into a single source of truth for 50 designers across web, iOS, Android, and email platforms.

Component Library
Overview

The Challenge

When I joined Bupa, the design organisation was working across 4 separate component libraries. Each platform—web, iOS, Android, and email—had evolved its own set of components, often with overlapping functionality but inconsistent implementation.

Designers were duplicating work. Development teams received different specs for the same patterns. Accessibility was inconsistent. And onboarding new designers meant teaching them 4 different systems.

The Problem

200+ Components, No Single Source of Truth

The numbers told the story:

  • 200+ components spread across 4 libraries
  • 75% duplication — the same patterns rebuilt multiple times
  • 50 designers across 8 squads with no shared language
  • Inconsistent accessibility — some components met WCAG, others didn't
  • No governance — anyone could add components without review

Approach

Audit, Consolidate, Govern

The solution wasn't just about reducing component count—it was about creating a sustainable system that would prevent fragmentation from happening again.

1. Component Audit

I mapped every component across all 4 libraries, identifying overlaps, inconsistencies, and gaps. This revealed that many "platform-specific" components were actually the same pattern with different names.

2. Core Component Definition

Working with development, I defined 50 core components that would serve as the foundation. Each component was designed to work across all platforms with appropriate adaptations, rather than platform-specific versions.

3. Contribution Model

I co-created a governance framework with clear approval flows. New components now require review, accessibility checks, and cross-platform consideration before being added to the system.

Collaboration

Working with Development

Consolidation only works if design and development are aligned. I established a weekly Design System sync with the development team to ensure:

  • Component specs matched technical feasibility
  • React and React Native implementations stayed aligned
  • Accessibility requirements were clear and testable
  • Changes were communicated before they landed

This ongoing collaboration meant fewer surprises, faster implementation, and a shared ownership of the system.

Outcome

A System Teams Can Rely On

The consolidated library became the single source of truth for all product design at Bupa. Results after 18 months:

  • 50 core components serving all platforms
  • 15+ contributions from 6 product teams through the new governance model
  • WCAG 2.2 compliance built into every component
  • Faster onboarding — new designers learn one system, not four
  • Reduced design debt — teams spend time on product problems, not rebuilding patterns
