Figma Linting Plugin

A custom plugin that automatically detects raw values in designs, catching 100+ issues monthly and improving design quality across the team.

Design Linting
Overview

The Challenge

With 50 designers across 8 squads, maintaining consistency is difficult. Even with a robust token system, designers sometimes reach for raw values—a specific hex colour instead of a token, a pixel value instead of a spacing token. These small deviations add up, creating drift between design and implementation.

Design reviews can catch some issues, but they're time-consuming and inconsistent. We needed automated quality control that could run before handoff.

The Problem

Raw Values Slipping Through

The issues we were seeing:

  • Hard-coded colours — hex values instead of colour tokens
  • Custom spacing — arbitrary pixel values instead of spacing scale
  • Non-standard typography — font sizes outside the type scale
  • Detached components — instances disconnected from the library
  • Inconsistent handoffs — developers receiving specs with raw values

Solution

Automated Design Linting

I built a Figma plugin that scans designs and flags any values that don't match our token system. It works like a code linter, but for design files:

!
Fill colour #0066CC → use color.action.primary
!
Spacing 18px → use spacing.16 or spacing.20
!
Font size 19px → use font.size.body (17px)
Features

What It Checks

Colour Tokens

Detects any fill or stroke that doesn't map to a token in our colour system.

Spacing Values

Flags padding, margins, and gaps that don't match our spacing scale.

Typography

Checks font sizes, weights, and line heights against the type scale.

Component Usage

Identifies detached instances and non-library components.

Workflow

Integrated Into Design Process

The plugin fits naturally into the design workflow:

During Design

Designers can run the linter at any point to check their work. Quick feedback helps catch issues early when they're easy to fix.

Before Handoff

A lint check before handoff ensures developers receive clean specs. No more back-and-forth about which token a colour should be.

Design Reviews

Reviewers can run the linter to quickly verify token compliance, freeing up review time for more substantive feedback.

Impact

Catching Issues Before They Ship

The plugin has become part of our standard workflow:

  • 100+ raw values caught monthly that would have made it to development
  • Faster handoffs — developers trust the specs they receive
  • Educational tool — designers learn the token system through feedback
  • Reduced review burden — automated checks handle the mechanical stuff
  • Better consistency — designs align with the system from the start
Looking Ahead

Native Figma Linting

Coming Soon: Figma's Native Variables Linting

Figma is building native variable linting features. When this ships, it will supersede our custom plugin—which is exactly the outcome we want. The plugin solved an immediate need, and now the platform is catching up. That's how good tooling should work.
